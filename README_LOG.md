# STM32F103 日志系统使用说明

## 概述

本日志系统专为STM32F103VET6设计，使用DMA+队列+FreeRTOS任务的形式，确保日志信息不会丢失。系统具有以下特点：

- **稳定可靠**：使用FreeRTOS队列缓冲日志消息，避免丢失
- **高效传输**：使用DMA进行串口传输，不占用CPU时间
- **多级别支持**：支持ERROR、WARNING、INFO、DEBUG四个级别
- **线程安全**：所有日志函数都是线程安全的
- **格式化输出**：支持printf风格的格式化输出

## 系统架构

```
应用程序
    ↓
LOG_XXX() 函数
    ↓
FreeRTOS 消息队列 (32个消息)
    ↓
日志任务 (高优先级)
    ↓
DMA传输缓冲区
    ↓
USART1 (115200波特率)
```

## 使用方法

### 1. 基本使用

```c
#include "log.h"

// 在main函数中初始化
int main(void) {
    // ... 其他初始化代码 ...
    
    // 初始化日志系统
    LOG_Init();
    
    // ... 其他代码 ...
}

// 在任务中使用
void MyTask(void *argument) {
    LOG_INFO("任务启动");
    LOG_ERROR("发生错误: %d", error_code);
    LOG_DEBUG("调试信息: %s", debug_string);
}
```

### 2. 日志级别

```c
LOG_ERROR("错误信息");      // 错误级别
LOG_WARNING("警告信息");    // 警告级别  
LOG_INFO("信息");           // 信息级别
LOG_DEBUG("调试信息");       // 调试级别
```

### 3. 格式化输出

```c
LOG_INFO("计数器: %d, 状态: %s", count, status);
LOG_ERROR("错误代码: 0x%08X", error_code);
LOG_DEBUG("浮点数: %.2f", float_value);
```

### 4. 原始字符串输出

```c
LOG_PrintRaw("不添加时间戳和级别的原始输出\r\n");
```

### 5. 日志级别控制

```c
// 设置最低日志级别（只显示ERROR和WARNING）
LOG_SetLevel(LOG_LEVEL_WARNING);

// 获取当前日志级别
log_level_t level = LOG_GetLevel();

// 获取队列中待处理的消息数量
uint32_t pending = LOG_GetPendingCount();
```

## 配置参数

在 `log.h` 中可以修改以下参数：

```c
#define LOG_MAX_MESSAGE_LENGTH    256     // 单条日志消息最大长度
#define LOG_QUEUE_SIZE           32      // 日志队列大小
#define LOG_TASK_STACK_SIZE     512      // 日志任务栈大小
#define LOG_TASK_PRIORITY       osPriorityHigh  // 日志任务优先级
```

## 硬件连接

- **USART1_TX**: PA9
- **USART1_RX**: PA10  
- **波特率**: 115200
- **DMA**: DMA1_Channel4

## 注意事项

1. **初始化顺序**：确保在FreeRTOS启动后初始化日志系统
2. **队列满处理**：当队列满时，新消息会被丢弃，不会阻塞系统
3. **DMA传输**：系统会自动等待DMA传输完成，确保数据完整性
4. **线程安全**：所有日志函数都是线程安全的，可以在任何任务中调用
5. **内存使用**：日志系统会占用约2KB的RAM（队列+缓冲区+任务栈）

## 故障排除

### 1. 日志不输出
- 检查串口连接
- 确认USART1初始化正确
- 检查DMA配置

### 2. 日志丢失
- 增加队列大小 `LOG_QUEUE_SIZE`
- 提高日志任务优先级
- 检查系统负载

### 3. 格式化错误
- 检查格式化字符串语法
- 确认参数类型匹配
- 避免缓冲区溢出
